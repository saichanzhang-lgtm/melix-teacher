## 三、推荐数据库架构

```
┌─────────────────────────────────────────────────┐
│                  前端 (HTML/JS)                    │
│              beijing-english-tutor/web/            │
└─────────────────────┬───────────────────────────┘
                      │ HTTP API
┌─────────────────────▼───────────────────────────┐
│              Node.js 服务层                        │
│              server.js (当前运行)                  │
│   • /api/chat      → AI对话                       │
│   • /api/health    → 健康检查                      │
│   • /api/config    → 配置信息                      │
└─────────────────────┬───────────────────────────┘
                      │
        ┌─────────────┼─────────────┐
        ▼             ▼             ▼
┌───────────┐ ┌───────────┐ ┌───────────┐
│ 文件知识库 │ │ RAG向量库  │ │ 结构化DB   │
│ (现有)     │ │ (建议接入) │ │ (建议接入) │
│           │ │           │ │           │
│ • .md文件 │ │ • 真题    │ │ • 词汇表  │
│ • 教材数据 │ │ • 范文    │ │ • 题库    │
│ • 考情    │ │ • 教案    │ │ • 学生档案│
│           │ │ • 教学资源│ │ • 试卷    │
└───────────┘ └───────────┘ └───────────┘
```

### 3.1 各层推荐技术栈

| 层级 | 当前方案 | 推荐升级方案 | 优先级 |
|------|----------|-------------|--------|
| 文件知识库 | .md 文件直接读取 | Markdown + 分段索引 | ✅ 当前可用 |
| RAG向量库 | 无 | ChromaDB / FAISS + Embedding | 🔴 高 |
| 结构化DB | 无 | SQLite (本地) / PostgreSQL (扩展) | 🟡 中 |
| 题库系统 | 无 | JSON文件库 / SQLite题库 | 🟡 中 |
| 词汇数据库 | 无 | SQLite词汇表 | 🟡 中 |
| 音频存储 | 无 | 本地MP3归档 + 索引 | 🟢 低 |

### 3.2 RAG接入方案（推荐优先实施）

```
用户提问
    ↓
Embedding 向量化
    ↓
ChromaDB 检索相关文档片段 (Top-K)
    ↓
拼接 System Prompt + 检索结果
    ↓
AI 模型生成回复
```

**建议接入步骤**：
1. 安装 ChromaDB：`npm install chromadb`
2. 准备文档 → 分块（chunk_size=500, overlap=100）
3. 使用 Embedding 模型（如 text-embedding-3-small 或本地模型）
4. 构建知识索引
5. 修改 server.js 的 `/api/chat` 路由，在调用 AI 前先检索
